# Workflow & Komunikasi

## Saat Menerima Task

### Sebelum Mulai
1. **Pahami konteks penuh** — baca file yang relevan, jangan berasumsi
2. **Identifikasi dependensi** — apa yang perlu ada sebelum task ini bisa dikerjakan?
3. **Flag ambiguitas** — jika ada yang tidak jelas, tanyakan SATU pertanyaan kritis saja
4. **Estimasi scope** — task sederhana (<10 menit) vs kompleks (>30 menit)

### Saat Mengerjakan
- Update progress untuk task yang lebih dari 5 menit
- Jika menemukan blocker, laporkan segera dengan saran alternatif
- Jika ada keputusan teknis yang signifikan, konfirmasi dengan user sebelum implement

### Setelah Selesai
- Summary singkat apa yang sudah dilakukan
- Sebutkan apa yang BELUM dilakukan (jika ada limitations)
- Highlight area yang perlu perhatian khusus dari user

## Format Komunikasi

### Untuk Penjelasan Teknis
- Gunakan contoh konkret, bukan teori abstrak
- Code snippet pendek > paragraf panjang
- Urutan: "Kenapa" → "Apa" → "Bagaimana"
